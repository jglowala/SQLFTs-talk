# RESUME INSTRUCTIONS

## 1. Identifying What Was Completed in the Previous Session
To determine what was completed in the last session:
- Check the project logs or commit history for any updates made before the token reset.
- Review any notes or documentation you may have kept during the previous work session.

## 2. Providing Context to Claude Code
When resuming the project, provide Claude Code with the following details:
- Summarize the main objectives you were working towards.
- List the files or components that were completed and those that are still in progress.
- Mention any specific challenges you faced or any coding decisions that were made previously.

## 3. If the PowerPoint File Was Partially Created
If the PowerPoint file is partially created:
- Locate the last saved version of the PowerPoint file to see what has been completed.
- Open the file and review the slides for any missing content or formatting that needs to be addressed.

## 4. Merging or Updating the Presentation
To merge or update the presentation if it was started:
- Use the latest version of the PowerPoint file as your base.
- Incorporate the changes made during previous sessions and ensure all content is aligned with the project goals.

## 5. Template for the Resume Prompt to use with Claude Code
When restarting with Claude Code, use the following template:
```markdown
**Resume Prompt**:

Context and Objective:
- [Insert context and main objectives]

Completed Items:
- [List completed items]

In Progress Items:
- [List in-progress items]

Challenges Faced:
- [Describe any challenges faced]

Next Steps:
- [Describe what needs to be done next]
```
